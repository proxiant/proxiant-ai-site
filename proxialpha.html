Control plane

A real multi-tenant backend. Not a hobby project.

ProxiAlpha ships with a FastAPI control plane backed by Postgres 16 with pgcrypto and row-level security. Every route is org-scoped, every write is audited, every usage event is metered.

Identity

Clerk SSO with JIT

Clerk-backed authentication with just-in-time user and org provisioning. JWKS keys rotate without downtime; a JTI replay cache blocks token reuse.

  • JWKS fetch + cache with graceful rotation
  • JTI replay protection on every request
  • Org resolution from Clerk claims, cached per-request
  • JIT user & org creation on first sign-in
Isolation

Postgres RLS & Alembic

Tenant isolation enforced at the database layer, not just the app. Every query runs under a session-level org_id GUC with row-level security policies on every tenant table.

  • RLS policies on users, agents, events, bundles, invoices
  • pgcrypto for at-rest secrets
  • Alembic migrations with up → down → up CI gate
  • Module-scoped async fixtures for fast schema tests
Billing

Stripe metered billing

Three entitlement types — consumables, caps, and feature flags — power a real metered billing pipeline. A Postgres advisory-locked cron reports usage to Stripe hourly, idempotently.

  • Atomic UPDATE … RETURNING consumable gates
  • Inline cap checks before expensive operations
  • Boolean feature flags for live trading and perps
  • Idempotent usage reporting keyed on (org, period, meter)
Agent fleet

Run the agent on your hardware. Keep the control plane in ours.

The ProxiAlpha agent is a small Python runtime you install on any Linux box. It phones home with JWT-authenticated heartbeats and can be revoked, rotated, or asked for a doctor bundle from the dashboard.

Enrollment

One-shot install tokens

Mint a single-use token from the dashboard, run the installer, and the agent exchanges it for a short-lived JWT signed by the control plane. Tokens are hashed at rest and consumed atomically.

  • Single-consume with 30-minute doctor grace window
  • Per-IP rate limiting on enrollment & heartbeat
  • Revocation flips a row — next heartbeat fails closed
  • Rotation without re-enrollment via refresh endpoint
Runtime

Heartbeat & health

Agents emit heartbeats on a fixed interval with version, host, and strategy status. The control plane tracks liveness per-host and surfaces stale agents in the dashboard and Prometheus metrics.

  • Structured last_seen_at per agent row
  • Version + strategy manifest reported each heartbeat
  • Automatic offline marking after miss threshold
  • Rate-limited; buckets per-IP and per-agent
Support

Doctor bundle ingest

When something goes wrong, the agent can package a diagnostic bundle (logs, config snapshot, last signals, system stats) and upload it to the control plane through a rate-limited, token-authenticated endpoint.

  • 5 per minute / 20 per hour per-IP bundle limits
  • Install-token auth with post-consume grace window
  • Stored under org_id with RLS-enforced access
  • Retention policy configurable per tier
